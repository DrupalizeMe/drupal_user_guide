=== Manually Downloading Module or Theme Files

[role="summary"]
How to manually download and upload a module or theme.

(((Module,downloading)))
(((Contributed module,downloading)))
(((Theme,downloading)))
(((Contributed theme,downloading)))
(((Downloading,module or theme files manually)))
(((Manually downloading module or theme files,overview)))

==== Goal

Laden Sie Modul- oder Themendateien manuell herunter und laden Sie sie auf Ihre Website hoch, wenn die
Website oder Drush-Methoden zur Installation oder Aktualisierung eines Moduls oder Themas nicht
Arbeit, oder wenn Sie ein speziell geschriebenes Modul oder Thema platzieren.

==== Erforderliche Vorkenntnisse

* <<understanding-modules>>
* <<extend-module-find>>
* <<understanding-themes>>
* <<extend-theme-find>>

==== Anforderungen an die Website

Für den Download des Handbuchs müssen Sie sich mit einem der folgenden Probleme auseinandersetzen
in diesem Thema beschrieben:

* Probleme mit der Dateierlaubnis

* FTP-Zulassungsprobleme

* Sie haben ein benutzerdefiniertes Modul/Thema erstellt oder seine Dateien von jemandem erhalten

* Sie konnten die Anweisungen in
<<extend-module-install>>, <<extend-theme-install>>,
<<security-update-module>>, oder <<security-update-theme>>

Überspringen Sie dieses Thema, wenn keiner der oben genannten Punkte auf Sie zutrifft. Wenn Sie den Composer verwenden
Um die Dateien auf Ihrer Website zu verwalten, folgen Sie den Schritten zur Dateiaktualisierung in
<<install-composer>> instead of this topic.

==== Steps

Wenn Sie ein Modul oder Thema von _Drupal.org_ installieren, folgen Sie den
Anweisungen zum Herunterladen und dann die Anweisungen zum Hochladen/Entpacken. Wenn
Wenn Sie das Modul oder Thema erstellt haben, überspringen Sie den Schritt des Herunterladens. Legen Sie stattdessen
eine Archivdatei (von der Sie wissen, wie man sie auf dem Server extrahiert) und fahren Sie fort mit
die Schritte des Hochladens/Entpackens, mit welcher Methode auch immer für die
wie Sie die Archivdatei ursprünglich angelegt haben.

==== Downloading the files

. Öffnen Sie das Modul oder Thema Projektseite auf drupal.org; zum Beispiel, die _Admin
Symbolleiste_ Seite (https://www.drupal.org/project/admin_toolbar).

. Scroll to the _Downloads_ section near the bottom of the page.
+
--
// Downloads section of the Admin Toolbar project page on drupal.org.
image:images/extend-manual-install-download.png["Downloading module files"]
--

. Klicken Sie auf _tar.gz_, um die Datei auf Ihren Computer herunterzuladen. Alternativ, wenn Sie
Terminal-Zugang zu Ihrem Hosting-Server (unter Linux) haben, können Sie die
Link-Adresse für die _tar.gz_-Datei von Ihrem Browser, und verwenden Sie diesen Befehl, um
die Datei herunterladen (wobei die kopierte URL ersetzt wird):
+
----
wget https://ftp.drupal.org/files/projects/admin_toolbar-8.x-1.26.tar.gz
----


==== Uploading the files to your site and unpacking them

. Wenn Sie ein neues Modul oder Thema hinzufügen, erstellen Sie Unterverzeichnisse in Ihrem
Top-Level-Verzeichnisse _Module_ und _Themen_ (falls sie nicht bereits
existieren). Typischerweise wird ein _Contrib_-Unterverzeichnis für beigetragene Module angelegt
und Themen, die von _Drupal.org_ heruntergeladen werden, sowie ein Unterverzeichnis _custom_ für
Module und Themen, die sie erstellt haben. Ihr _Module_-Verzeichnis könnte wie folgt aussehen
dies:
+
--
// Make custom and contrib directories under modules, and take a screenshot
// showing the directory structure.
image:images/extend-manual-install-directories.png["Suggested directory structure for modules",width="174px"]

// NOTE for Translators: you don't need to localize the 'custom' and 'contrib' directory names as they are more common in English.
--

. Wenn Sie ein vorhandenes Modul oder Thema durch eine aktualisierte Version ersetzen wollen, setzen Sie
die Website in den Wartungsmodus. Siehe <<Erweiterte Wartung>>>.

. Wenn Sie ein vorhandenes Modul oder Thema durch eine aktualisierte Version ersetzen, finden Sie
und löschen Sie alle vorhandenen Dateien und Verzeichnisse für das vorhandene Modul
oder Thema. Die Module befinden sich normalerweise in Verzeichnissen unter der obersten Ebene
_Module_-Verzeichnis, und Themen befinden sich normalerweise in Verzeichnissen unter dem
Verzeichnis der obersten Ebene _Themen_.

. Laden Sie die _.tar.gz_-Datei (oder welches Archiv auch immer Sie erstellt haben) auf Ihre Website hoch.
Platzieren Sie sie entweder an demselben Ort, von dem aus Sie das Verzeichnis gelöscht haben
(wenn ein bestehendes Modul oder Thema ersetzt wird) oder das entsprechende Unterverzeichnis
von _Modulen_ oder _Themen_ (wenn ein neues Modul oder Thema hinzugefügt wird).

. Extrahieren Sie die Dateien aus dem _.tar.gz_-Archiv (oder dem Archiv, das Sie
erstellt), wobei ein Unterverzeichnis am gleichen Ort wie die Archivdatei erstellt wird.
Wenn Sie keinen Terminal-Zugang haben oder Ihr Hosting-Server nicht läuft
Linux, der Dateimanager Ihres Hosting Control Panels sollte eine Möglichkeit bieten
die Dateien extrahieren. Wenn Sie Terminal-Zugang zu Ihrem Hosting-Server haben
(unter Linux) und Sie eine tar.gz-Datei verwenden, können Sie einen Befehl wie
+
----
tar -xzf admin_toolbar-8.x-1.17.tar.gz
----

. Löschen Sie die komprimierte Datei vom Server, es sei denn, Ihre Entpackmethode
bereits gelöscht.

. Siehe <<extend-module-install>>, <<extend-theme-install>>,
<<security-update-module>>, oder <<security-update-theme>> zur Vervollständigung der
Installation oder Aktualisierung des Moduls oder Themas. Beginnen Sie mit dem Schritt nach der
Der automatische Download ist abgeschlossen.

==== Vertiefen Sie Ihr Verständnis

* Wenn Sie mit mehreren Umgebungen arbeiten (z.B. eine lokale Entwicklungsumgebung
und einer Produktionsstätte) müssen Sie die Schritte für jede Umgebung wiederholen, oder
die Umwelt reklonieren. Siehe <<<installieren-entwickeln>>.

* Wenn Sie ein neues Thema hinzugefügt haben, navigieren Sie im Verwaltungsmenü _Verwalten_ zu
_Erscheinung_ (_Admin/Erscheinung_) und deinstallieren Sie das alte Thema.
