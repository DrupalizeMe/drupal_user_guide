==== Was ist modularer Inhalt?

Da der Inhalt Ihrer Website in einer Datenbank gespeichert ist, ist es
wünschenswert die Inhalte Ihrer Website _modular_ aufzubauen, d.h. bestimmte
Seiten auf Ihrer Website, einzeln zu bearbeiten, statt jedes mal die gesamte
Website zu editieren. Drupal holt sich den Rest der Elemente, die notwendig sind
um eine komplette Website zusammenzusetzen aus anderen Inhaltselementen.
Im Szenario der Wochenmarkt-Website für Landwirte könnten Sie zum Beispiel
einzelne Inhaltselemente für Rezepte anlegen. Jede einzelne Zutat eines Rezepts
entspricht dabei einem Feld, das die jeweilige Zutat enthält. So kann Ihre
Website die notwendigen Zutaten für ein bestimmtes Rezept als solches anzeigen
und die Besucher Ihrer Website können die Rezepte auf Ihrer Website nach
Zutaten durchsuchen, die Sie auf dem Wochenmarkt gekauft haben.

Kleinere Seitenabschnitte können auch zusammengesetzt werden  Rezepte können
beispielsweise ein Feld enthalten, in dem die Information gespeichert wird,
welcher Lieferant das jeweilige Rezept eingereicht hat .
Siehe: <<structure-reference-fields>>).
Diese Informationen über den Lieferanten können dann in einem separaten
Inhaltstyp bearbeitet werden. So können Sie auf Ihrer Website zum Beispiel
folgendes tun:

* Auf jeder Rezeptseite könnte es einen Bereich geben, der einige Informationen
darüber anzeigt, welcher Lieferant das Rezept eingereicht hat und zusätzliche
Metainformationen zum Lieferanten, wie z.B. dessen Name und die Nummer seines
Stands auf dem Wochenmarkt ausgeben.

* Jede Lieferantenseite könnte einen Abschnitt enthalten, der alle Rezepte
enthält, die der jeweilige Lieferant eingereicht hat.

Der Kerngedanke ist, dass jede Information nur an einer Stelle bearbeitet wird.
Wenn die Informationen eines Lieferanten aktualisiert werden, werden alle
Rezeptseiten, die Informationen über den Lieferanten, auf allen Rezeptseiten,
die der Lieferant angelegt hat, automatisch mit aktualisiert.
Die Informationen werden auch automatisch angepasst, wenn ein Lieferant ein neues Rezept einreicht.
Das mit dem Kernsystem ausgelieferte Modul Views ist das Modul, mit dem Sie
üblicher weise solche modularen Inhalte erstellen.
Mit Views können Sie zusammengesetzte Seiten und zusammengesetzte
Seitenabschnitte erstellen.
Siehe: <<<views-concept>> für weitere Informationen. Außerdem sind
_Ansichtsmodi_ nützlich um definieren zu können, auf welche Art und Weise
Inhalte angezeigt werden sollen und ob es für ein Inhaltselement mehrere
Anzeigearten geben soll.
Siehe <<structure-view-modes>> für weitere Informationen.
