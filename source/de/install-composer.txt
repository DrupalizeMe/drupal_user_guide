Zielsetzung

Verwenden Sie den Composer, um Dateien und Abhängigkeiten in der Kern-Software herunterzuladen oder zu aktualisieren,
oder in Zusatzmodulen und -themen. Überspringen Sie dieses Thema, wenn Sie den Composer nicht verwenden.

==== Erforderliche Vorkenntnisse

<<install-tools>>

==== Voraussetzungen für den Standort

Wenn Sie den Composer verwenden möchten, muss er entweder auf einer lokalen Entwicklung
Server oder Ihre Live-Site. Siehe <<installierte Werkzeuge>>.

==== Steps

Wenn Sie das Tool Composer nicht auf Ihrem Live-Server installieren können, können Sie
folgen Sie den Schritten in einem der folgenden Abschnitte auf Ihrem lokalen Server, und dann
alle aktualisierten oder hinzugefügten Dateien auf Ihren Live-Server übertragen. Die empfohlene
Das Verfahren besteht darin, ein Archiv oder eine Zip-Datei der neuen und geänderten Verzeichnisse zu erstellen,
übertragen Sie das Archiv auf Ihren Live-Server, löschen Sie die Verzeichnisse, die
geändert, und extrahieren Sie das Archiv. Stellen Sie sicher, dass Sie nach Aktualisierungen und Ergänzungen suchen.
auf die folgenden Dateien im Stammverzeichnis Ihrer Installation:

* _vendor_ directory
* _autoload.php_
* _composer.json_
* _composer.lock_

==== Using Composer to download the core software

Folgen Sie diesen Schritten, wenn Sie den Kern noch nicht heruntergeladen oder installiert haben
Software, und Sie möchten Composer verwenden, um sowohl die Kern-Software als
seine externen Abhängigkeiten:

. Wechseln Sie auf der Kommandozeile in eine Ebene über dem Verzeichnis, in dem Sie
die Software, die sich dort befinden soll.

. Geben Sie diesen Befehl ein, wobei _mydir_ das Verzeichnis ist, das Sie erstellen möchten:
+
----
composer create-project drupal/recommended-project mydir
----

. Die neueste Version der Kern-Software wird auf das _mydir/web_ heruntergeladen.
Unterverzeichnis.

======= Konvertieren einer zuvor heruntergeladenen Site zur Verwendung von Composer

Wenn Sie zuvor die Kern-Software heruntergeladen haben, ohne Composer zu verwenden, können Sie
später feststellen, dass Sie den Composer zur Verwaltung und Aktualisierung verwenden wollen oder müssen
Abhängigkeiten. Für Sites, die mit Version 8.8.0 oder höher des Kerns erstellt wurden
Software enthalten die Dateien bereits die erforderliche Composer-Konfiguration und Sie
können jederzeit mit der Nutzung des Composers beginnen.

Wenn Ihre Website Module, Themen oder Profile enthält, die nicht Teil der
Kern-Software ist es eine gute Idee, sie in die Datei _composer.json_ in der
Stammverzeichnis Ihrer Website.

Führen Sie den folgenden Befehl einmal für jedes heruntergeladene Modul, Thema oder Profil aus
von Drupal.org, wobei der Name des gewünschten Projekts durch +module_name+ ersetzt wird
hinzufügen:

----
Komponist benötigen drupal/module_name
----

Weitere Informationen zur Verwaltung von Site-Abhängigkeiten mit Composer finden Sie weiter unten.

So konvertieren Sie eine Site, die mit einer Version der Kern-Software vor
8.8.0, oder das von einer Zip- oder tar.gz-Datei installiert wurde, die von
Drupal.org, siehe die Anweisungen unter
https://www.drupal.org/docs/8/install/add-composer-to-an-existing-site[Komponist zu einer bestehenden Website hinzufügen].

====== Composer zum Herunterladen eines Moduls oder Themas verwenden

Befolgen Sie diese Schritte, wenn Sie bereits Composer zur Verwaltung des Kerns verwenden
Software, und Sie möchten Composer verwenden, um ein beigetragenes Modul oder Thema hinzuzufügen mit
seine Abhängigkeiten.

. Jedes Mal, wenn Sie ein beigetragenes Modul oder Thema hinzufügen möchten, bestimmen Sie die
den Kurznamen des Projekts. Dies ist der letzte Teil der URL der Projektseite; für
Beispiel, das Modul Geofield, unter
https://www.drupal.org/project/geofield, hat den Kurznamen +geofield+.

. Zum Herunterladen des beigetragenen Moduls oder Themas, zusammen mit seinem externen
Abhängigkeiten, geben Sie den folgenden Befehl im Stammverzeichnis Ihrer Website ein (ersetzen Sie
die Kurzbezeichnung des Moduls oder Themas für +geofield+):
+
----
Komponist benötigen Drupal/Geofield
----

====== Composer verwenden, um zuvor heruntergeladene Dateien zu aktualisieren

Befolgen Sie diese Schritte, um die Dateien für die Kern-Software oder eine beigesteuerte
Modul oder Thema, nachdem bereits begonnen wurde, Abhängigkeiten zu verwalten mit
Komponist:

. Bestimmen Sie die Kurzbezeichnung des Projekts, das Sie aktualisieren möchten. Für den Kern
Software, es ist _Kern_. Für beigetragene Module und Themen ist es das letzte
Teil der URL der Projektseite; zum Beispiel das Geofield-Modul, unter
https://www.drupal.org/project/geofield, hat den Kurznamen +geofield+.

. Wenn Sie auf das neueste stabile Release aktualisieren möchten, verwenden Sie
Befehl, wobei der Kurzname des zu aktualisierenden Projekts ersetzt wird durch
+geofield+:
+
----
komponist update drupal/geofield --mit-abhÃ?ngigkeiten
----

. Wenn Sie eine bestimmte Version benötigen, legen Sie fest, wie Sie die Versionsnummer eingeben
aktualisieren möchten. Zum Beispiel für Version 8.x-1.7 eines beigetragenen Moduls
würde nur die Version 1.7 eingeben, und für die Kern-Software-Version 8.3.1 würden Sie
8.3.1 eingeben. Geben Sie dann den folgenden Befehl im Stammverzeichnis Ihrer Website ein
(Ersetzen des Kurznamens des Projekts durch +geofield+ und die korrekte
Versionsnummer):
+
----
Komponist benötigen Drupal/Geofield:1,7
----

==== Erweitern Sie Ihr Verständnis

Sie können mehr über die Composer-Befehle erfahren, indem Sie die integrierte Hilfe von Composer verwenden
System. Um beispielsweise mehr über den Befehl `Projekt erstellen` zu erfahren, geben Sie
`composer help create-project` in Ihrem Befehlsfenster.
